# Module 2: Data Cleaning & Feature Engineering Specification


## 1. Objective


- **Purpose**: Transform raw financial data into a clean, consistent, and feature-rich dataset ready for visualization and AI analysis.
- **Input**: Raw CSV files generated by Module 1.
- **Output**: A processed dataset with comprehensive technical indicators, momentum oscillators, and risk metrics.


## 2. Data Cleaning Requirements


Implement a pipeline that handles the following:


- **Missing Values**: Use a "Forward-fill" strategy (ffill) followed by "Backward-fill" (bfill) for price series.
- **Duplicate Records**: Detect and remove duplicate entries based on `date` and `ticker`, with logging for any removed rows.
- **Data Normalization**:
 - Ensure the `date` column is converted to `datetime` objects and sorted in ascending order.
 - Standardize all numeric columns to float types.
- **Outlier Handling**: Identify and flag anomalies, specifically accounting for data errors or unadjusted stock splits (using IQR method).




## 3. Fundamental indicator
### A. Profitability ratio
- **ROA** = Take `roa` collected from Module 1 
- **ROE** = Take `roe` collected from Module 1  
- **Gross Profit Margin** = `gross_profit` / `revenue` 
- **Net Profit Margin** = `net_income` / `revenue`
- Revenue_growth = (`revenue_current` - `revenue_previous`) / `revenue_previous`


### B. Activity ratio


- Receivable Turnover = `revenue` / avg(`receivables`)


- Inventory Turnover = `cogs` / avg(`inventory`); where `cogs` = `revenue` ‚àí `gross_profit`


- Payable Turnover = `purchase` / avg(`payables`); where `purchase` = `ending inventory` ‚àí `beginning inventory` + `cogs`


- Cash Conversion Cycle (CCC) = 365 / `Receivable Turnover` + 365 / `Inventory Turnover` ‚àí 365 / `Payable Turnover`
### C. Liquidity & Solvency ratio
- Current Ratio = `current_assets` / `current_liabilities`
- D/E (Debt-to-Equity) = `total_debt` / `equity`
- Interest Coverage = `operating_profit` / `interest_expense`






### D. Cash flow
- **FCFF (Free Cash Flow to Firm)** = `operating_cash_flow` + `interest_expense` √ó (1 ‚àí `tax_rate`) ‚àí `capital_expenditure` 
- **FCFE (Free Cash Flow to Equity)** = `operating_cash_flow` ‚àí `capital_expenditure` + `net borrowing`; where net borrowing = change in `total_debt`
### E. Valuation Metrics
- **P/E 1Y average**: Rolling 252-day mean of `pe` ratio 
- **P/E 5Y average**: Rolling 1260-day mean of `pe` ratio 
- **P/E industry**: Pass through from `industry_df` (`industry_pe`) 
- **P/B 1Y average**: Rolling 252-day mean of `pb` ratio 
- **P/B 5Y average**: Rolling 1260-day mean of `pb` ratio 
- **P/B industry**: Pass through from `industry_df` (`industry_pb`) 
- **DCF Intrinsic Price (FCFE method)**: 
- `Ke` = `risk_free_rate` + `beta` √ó `market_risk_premium` 
- Growth rate `g` = trailing 3-year `FCFE` growth rate 
- `Terminal Value` = `FCFE_last` √ó (1+`g`) / (`Ke` ‚àí `g`) 
- `Equity Value` = Œ£ [`FCFE_t` / (1+`Ke`)^t] + `Terminal Value` / (1+`Ke`)^n 
- `Intrinsic Price` = `Equity Value` / `shares_outstanding` 
- `Upside/Downside` = (`Intrinsic Price` ‚àí current `close`) / current `close`




## 4. Feature Engineering (Technical Indicators)


Calculate the following features for each ticker using `pandas` and the `ta` (Technical Analysis) library:


### A. Return Metrics


- **Daily Return**: $r_t = \frac{P_t - P_{t-1}}{P_{t-1}}$
- **Log Return**: $ln(P_t / P_{t-1})$
- **Cumulative Return**: Over 7-day, 30-day, 90-day, and Year-To-Date (YTD).


### B. Trend Indicators & Momentum Oscillators


- Moving Averages (MA): MA20, MA50, MA200.
Signal logic: If close > MA ‚Üí BUY, if close < MA ‚Üí SELL. Apply for both SMA and EMA.
- RSI (Relative Strength Index, 14-day): Identify overbought (>70) and oversold (<30) conditions.
- MACD (12,26): MACD line (EMA12 - EMA26), Signal line (EMA9 of MACD), and MACD Histogram.
- Bollinger band: 
MB = (P1 + P2 + ... + P20) / 20

UB = MB + (2 √ó œÉ)
LB = MB - (2 √ó œÉ)




### C. Price and volume anomalies
- Volume Spike: Flag days where volume > 2√ó rolling 20-day average volume. Indicates unusual trading activity.
- Gap Up/Down: Flag days where open > previous high (gap up) or open < previous low (gap down). Indicates sudden market sentiment shift.
- Sudden Price Movement: Flag days where |daily_return| > 3√ó rolling 20-day standard deviation of returns. Indicates abnormal price action.


### E. Risk Metrics
- Historical Volatility: Rolling 30-day and 60-day standard deviation of daily returns.
- Beta: 60-day rolling beta vs market benchmark (VN-Index for VN market, S&P500 for Global).
- VaR (Value at Risk) 95%: 5th percentile o·ªùrolling daily returns distribution.
- VaR (Value at Risk) 99%: 1st percentile of rolling daily returns distribution.
- Max Drawdown: Peak-to-trough decline.
- Sharpe Ratio: Annualized return divided by annualized volatility.




## 5. Multi-Asset Features
- **Correlation Matrix**: Pairwise correlation of daily returns between selected assets (stock vs. benchmark vs. peers).
- **Relative Strength (RS Line)**: Ratio of stock cumulative return to benchmark cumulative return, used to identify market outperformers.


## 5.1 Visualization-Ready Features (Current Project)


In the current implementation, the visualization stage uses processed indicator columns to render:


- **Returns Distribution**: Histogram + KDE (with fallback smoothed density line when KDE is unstable).
- **Risk Marker Lines**: `Return = 0`, `Mean`, `Median`, `VaR 95%`, `VaR 99%`.
- **Correlation Heatmap**: Correlation across selected indicators for the target ticker.
- Efficient frontier: Generate a range of portfolio weights between Stock A and Stock B to calculate simulated returns and volatility for optimization plotting.
